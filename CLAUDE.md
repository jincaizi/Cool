## Project Type

This is a Unity 3D game development project using Unity 2022.3.25f1.

## Commands

### Build and Run

- Open the project in Unity Editor: Open Unity Hub and select this project folder, or run `Unity.exe -projectPath "E:\CodeForJob\Cool"`
- Build the project: Use Unity Editor's Build Settings menu (File > Build Settings)
- Run the game: Press Play in Unity Editor

### Testing

- Run tests in Unity Editor: Open the Test Runner window (Window > General > Test Runner) and run tests from there
- Command line test execution (requires Unity installation):
  ```bash
  Unity.exe -batchmode -projectPath "E:\CodeForJob\Cool" -runTests -testPlatform EditMode
  Unity.exe -batchmode -projectPath "E:\CodeForJob\Cool" -runTests -testPlatform PlayMode
  ```

### Code Generation

- Generate Visual Studio solution: Unity automatically generates `.csproj` and `.sln` files when scripts are modified. You can also manually regenerate by opening the project in Unity and waiting for compilation to complete.

### Clean Project

- Delete Library, Temp, Logs, and obj folders to force Unity to reimport all assets:
  ```bash
  rm -rf Library Temp Logs obj
  ```

## Project Structure

```
Cool/
├── Assets/           # All game assets (scenes, scripts, prefabs, etc.)
│   └── Scenes/       # Unity scene files
├── Packages/         # Unity package manifest
├── ProjectSettings/  # Unity project configuration
└── Library/          # Auto-generated by Unity (gitignored in practice)
```

## Dependencies

Managed via `Packages/manifest.json`:
- **com.unity.test-framework** (1.1.33) - Unity Test Framework with NUnit support
- **com.unity.textmeshpro** (3.0.6) - Text rendering system
- **com.unity.timeline** (1.7.6) - Cinematics and sequencing
- **com.unity.ugui** (1.0.0) - Unity UI system
- **com.unity.visualscripting** (1.9.2) - Visual scripting system

## Important Notes

- The `Library/` folder contains auto-generated files from Unity's Package Manager. These files should not be directly edited.
- The project uses Assembly Definition files (`.asmdef`) for organizing code into assemblies.
- Unity automatically compiles C# scripts in the `Assets/` folder.

## Testing

This project includes the Unity Test Framework. Tests are organized into:
- **Edit Mode Tests**: Run outside of Play Mode, for testing editor and non-gameplay logic
- **Play Mode Tests**: Run in Play Mode, for testing gameplay and runtime behavior

## Development Workflow

1. Make changes to scripts in the `Assets/` folder
2. Unity Editor will automatically detect changes and recompile
3. Test changes by running the game in Play Mode
4. Commit changes to git (excluding auto-generated files like Library/, Temp/, Logs/)
